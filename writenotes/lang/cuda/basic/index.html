    #include<stdio.h>

__global__ void add(int a, int b, int *c){
    *c = a + b;
}

int main(){
    int c;

    // Allocate Unified Memory - accessible from CPU or GPU
    int* c_cuda;    // a memory for a int data
    cudaMalloc((void**)&c_cuda, 1*sizeof(int)); //return pointer

    // Launch kernel on 1 block containing 1 thread
    add<<<1,1>>>(2, 3, c_cuda);

    // Transfer data between GPU VRAM and CPU RAM
    cudaMemcpy(&c, c_cuda, sizeof(int), cudaMemcpyDeviceToHost);
    printf("c=%d\n", c);

    // Free the allocated unified memory
    cudaFree(c_cuda);
    return 0;
}

  3. Adapt C to CUDA (kernel function):

    1. The function add is declared as a __global__ function, and then it becomes a kernel, which is called from CPU and executed on GPU.

    2. Kernels have no return value, as results are left in memory.

      • Thus, the GPU memory for output data must be allocated (by cudamalloc) and passed into the kernel.
      • And free it (with cudaFree) at the end.

    3. A kernel needs execution configuration: <<<blocks, threads, shared_mem, stream>>> to sequentialize the host code before host compilation. Docs

Blocks & Threads

  • CUDA locates a thread via blocks.

  • Number of threads in a block is a multiple of 32, e.g., 256 threads. An Even Easier Introduction to CUDA - Nvidia Blog - Mark Harris, Jan25, 2017

  • Total threads can be reshaped to 2D or 3D, and accordingly the kernel needs to modify the threads indexing.

  • Number of blocks must be larger than the total elements.

    Given N elements, there needs n = (N + blkDim.x - 1)/blkDim.x

  • Index of a thread is blockIdx.x * blockDim.x + threadIdx.x.

    And a grid includes all the threads = gridDim.x * blockDim.x. (grdiDim in the following figure may be wrong.)
