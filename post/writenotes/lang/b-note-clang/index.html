Stack Memory

  1. 压栈

    • Supports:

      (2025-07-02T15:26)

      1. RL78 的内存划分

      2. static 有固定的地址，它们不会在程序运行时被销毁。 局部变量占用的内存在栈区，程序执行完毕后，它们在栈中的空间会被清除（出栈）， 所以局部变量在函数每次执行时，都会在栈中重新分配地址，所以地址不一样。

      3. 当要调用 func1 的时候，需要把它所需使用的资源放入 stack

      4. 压栈顺序：

        1. 返回地址（即 func1 的下一个函数的地址），

        2. 传入参数（如果是 primitive 变量直接把值放入栈中， 如果是数组，就把数组第一字节的地址放入栈中）

        3. 返回值（只是留出空间？因为还没算出结果？）

        4. 局部变量只是给它们在栈里留出空间（也有的 compiler 会分配固定地址）

      5. 栈的使用是动态变化的，因为同一时间，只能有一个函数运行 （严格来讲是只有一条指令运行，除非是双核 cpu，那就需要处理资源调度问题） 如果有中断发生，那么 ISR 的变量（返回地址，传入变量，返回值，局部变量）就需要压栈， 如果中断太多，可能就会发生 stack overflow

      6. 被 static 修饰的局部变量被放在和全局变量一起，不在栈中，所以不会被销毁

      7. 8 位处理器返回一个 16 位数据时，需要 2 次操作，这两步操作不能被打断，需要加锁
